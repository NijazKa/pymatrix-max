# Архитектура

## Компоненты

```text
Element / Matrix client
        │
        ▼
      Synapse
        │ Application Service transactions
        ▼
   pymatrix-max
        │ one PyMax client per Matrix user
        ▼
       MAX
```

## Пользовательские сессии

Каждому Matrix-пользователю с уровнем `user` или `admin` соответствует отдельный
`User.max_client`. MAX-сессии хранятся в раздельных каталогах. Мост сохраняет
стабильный профиль устройства, чтобы модель Android и идентификаторы не менялись
при каждом рестарте.

## Личные чаты

DM portal связан с владельцем MAX-сессии и удалённым MAX-контактом. Matrix-комната
содержит реального Matrix-пользователя и ghost контакта `@max_<id>:domain`.
Bridge bot не должен становиться третьим участником DM.

## Группы и каналы

Группа или канал используют общий portal по `chat_id`. Таблица `portal_user` хранит,
какие Matrix/MAX-аккаунты состоят в удалённом чате. При включённом
`group_messages_via_bot` сообщения отправляет bridge bot, а исходные MAX ID и имя
автора сохраняются в `message_map`.

Это позволяет:

- приглашать нескольких Matrix-пользователей в одну общую комнату;
- корректно проверять право отправки;
- поддерживать ответы и `contact`;
- не регистрировать ghost для каждого участника большой группы.

## Сопоставление сообщений

`message_map` хранит MAX message ID и Matrix event ID. Таблица используется для:

- двусторонних replies;
- дедупликации и подавления эхо;
- привязки реакций;
- определения автора relay-сообщения.

## Реакции

Matrix → MAX использует MAX reaction API и `reaction_map`. Обратное направление
зависит от входящего push-события PyMax. Протокол групповых реакций может не
содержать ID автора, поэтому точное отображение ограничено.

## Медиа

- обычные файлы скачиваются во временную память с контролем размера;
- MIME определяется по заголовкам и сигнатуре;
- MAX может вернуть MPEG-DASH manifest вместо MP4;
- FFmpeg скачивает DASH-сегменты и выполняет remux `-c copy` в MP4;
- итог загружается в Matrix Content Repository.

## Блокировка чатов

`blocked_chat` — локальный персональный denylist. Polling отсутствует: повторный
выход инициируется только при событии MAX для заблокированного чата и защищён от
частых повторов.
