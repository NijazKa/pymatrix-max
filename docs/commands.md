# Команды bridge bot

Во всех примерах используется префикс `!max`. В management-комнате mautrix может
также принимать команду без префикса, но для однозначности документация всегда
показывает полный вариант.

## Авторизация

### `!max login <телефон>`

Запускает вход в MAX. Телефон указывается в международном формате.

```text
!max login +79991234567
```

Код и пароль 2FA отправляйте обычными сообщениями в ответ на запрос бота.

### `!max cancel`

Отменяет незавершённый login flow.

### `!max logout`

Завершает удалённую MAX-сессию и очищает login token. Стабильный профиль устройства
сохраняется для следующего входа на тот же номер.

### `!max ping`

Показывает состояние моста и готовность MAX-клиента.

## Контакты

### `!max add <телефон>`

Находит контакт MAX по телефону и создаёт или открывает личную Matrix-комнату.

```text
!max add +79991234567
```

### `!max contact [MAX ID]`

Показывает имя, MAX ID и телефон, если MAX его предоставляет. В личном чате аргумент
не нужен. В группе можно ответить этой командой на сообщение нужного автора.

## Группы и каналы

### `!max join <ссылка>`

Вступает в группу или канал MAX по invite/public ссылке и создаёт Matrix portal.

### `!max syncgroups [chat_id или Matrix room_id]`

Сверяет известные группы и приглашения. Без аргумента используется список чатов,
полученный MAX-клиентом. С аргументом выполняется один точечный запрос.

### `!max leave [chat_id]`

Выходит из удалённой MAX-группы или канала. Выход из комнаты в Element сам по себе
не покидает MAX-чат. После успешной команды покиньте Matrix-комнату средствами
Element.

Повторный `leave` для уже покинутого чата выполняет локальную очистку и не считается
ошибкой.

### `!max block [chat_id]`

Выходит из MAX-чата и сохраняет персональный denylist. При реальном повторном
добавлении мост событийно попытается сразу выйти и не будет пробрасывать чат.

### `!max unblock [chat_id]`

Удаляет запись denylist.

### `!max blocked`

Показывает заблокированные MAX-чаты.

## Диагностика

### `!max debug`

Показывает текущие asyncio-задачи login flow. Вывод может содержать технические
детали; не прикладывайте его к публичному issue без проверки.
